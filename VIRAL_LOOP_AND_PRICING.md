### 1.4 레퍼럴 시스템 상세

#### 레퍼럴 코드 생성

```php
class JJ_Referral_System {

    // 코드 생성
    public static function generate_code($user_id) {
        $hash = substr(md5($user_id . get_option('jj_referral_salt')), 0, 8);
        return 'JJ-' . strtoupper($hash);
    }

    // 레퍼럴 URL
    public static function get_referral_url($user_id) {
        $code = self::generate_code($user_id);
        return add_query_arg('ref', $code, home_url('/pricing/'));
    }

    // 레퍼럴 추적
    public static function track_referral($code) {
        $referrer = self::get_user_by_code($code);
        if ($referrer) {
            // 쿠키 설정 (30일)
            setcookie('jj_referral', $code, time() + (30 * DAY_IN_SECONDS), '/');

            // 방문 로그
            self::log_visit($code);
        }
    }

    // 보상 지급
    public static function process_reward($new_user_id, $purchase_amount = 0) {
        $code = $_COOKIE['jj_referral'] ?? '';
        if (!$code) return;

        $referrer_id = self::get_user_by_code($code);

        if ($purchase_amount > 0) {
            // 유료 전환: 20% 수익 공유
            $reward = $purchase_amount * 0.20;
            self::add_credit($referrer_id, $reward);

            // 피추천인 할인은 결제 시 이미 적용됨
        } else {
            // 무료 가입: 포인트 적립
            self::add_points($referrer_id, 100);
            self::add_points($new_user_id, 50);
        }

        // 이메일 알림
        self::notify_referrer($referrer_id, $new_user_id, $purchase_amount);
    }
}
```

### 1.5 K-Factor 목표 및 측정

#### K-Factor 공식

```
K = i × c

i = 사용자당 평균 초대 수
c = 초대의 전환율

목표:
- i = 3 (사용자당 3명 초대)
- c = 40% (초대받은 사람 중 40% 가입)
- K = 3 × 0.4 = 1.2

K > 1 = 바이럴 성장
```

#### 측정 대시보드

| 지표 | 정의 | 목표 |
|------|------|------|
| 초대 수 (i) | 월평균 사용자당 초대 | 3+ |
| 전환율 (c) | 초대 → 가입 | 40% |
| K-Factor | i × c | 1.2+ |
| 바이럴 사이클 시간 | 초대 → 가입 평균 시간 | 3일 이하 |
| 공유율 | MAU 중 공유한 비율 | 15% |
