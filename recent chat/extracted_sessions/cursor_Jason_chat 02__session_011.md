**Cursor**

### 제가 추천하는 다음 계획(순서 고정, “근본 원인 → UX → 운영”)

- **1) 무오류/안정성 ‘회귀 방어막’ 먼저 강화**
  - **목표**: 다시는 `Undefined function/constant`, 누락 파일 include, 로드 순서 문제로 사이트가 죽지 않게 만들기
  - **작업**: 핵심 상수/헬퍼를 *가장 먼저* 로드되도록 부트스트랩 정리, `file_exists/class_exists/defined` 기반 안전 로더 표준화, “필수 파일/뷰 존재” 매니페스트 점검, 관리자에서 **Self-test(활성화/업데이트/팔레트/AI 프리셋)** 확장

- **2) Admin Center > Updates 탭을 ‘워드프레스 플러그인 목록 UX’로 재구성**
  - **목표**: 코어/애드온 전체를 한 화면에서 “상태/자동업데이트/업데이트 가능/채널”이 직관적으로 보이게
  - **UI**: WP 목록처럼 **테이블 + 배지(채널/자동업데이트/업데이트 있음) + 즉시 액션(체크/토글/로그 보기)**  
  - **원칙**: **Master/Partner는 업그레이드 유도/제한 메시지 0**, 채널 정책(예: Master는 beta off 등)은 중앙에서 일관 적용

- **3) 추천 팔레트/AI 프리셋의 ‘적용/검증/되돌리기’ 운영 UX 완성**
  - **목표**: 실사용자가 “적용 → 결과 확인 → 필요 시 즉시 되돌림”을 빠르게
  - **작업**: 적용 전/후 미리보기(요약 diff), Undo(임시 저장 기반), 프리셋 태그/정렬/정리 UX 마감

- **4) 배포/런처 시스템을 ‘팀 운영 도구’로 한 단계 업그레이드**
  - **목표**: 누가/언제/무엇을 배포했는지 + 무엇이 바뀌었는지(핵심 diff) 한 번에 추적
  - **작업**: 대시보드에 변경 요약/파일 diff 링크, 릴리즈 노트 자동 생성, 롤백 ZIP 바로가기

### 바로 다음 라운드 “첫 착수”는 이렇게 가겠습니다
**(1) 안정성 회귀 방어막 강화 → (2) Updates 탭 WP-리스트형 재구성** 순으로 들어가겠습니다.
